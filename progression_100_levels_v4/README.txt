Файл progression.json — таблица прокачки v5 (уровни в игре 1…100).

Поля:
  xp_to_next       — длина 99; индекс i = переход с ур.(i+1) на ур.(i+2), т.е. 1→2 … 99→100.
  xp_per_win       — длина 100; опыт за победу на ур. 1…100.
  steps_per_level  — длина 99; сколько «апов» (промежуточных +1 свободного стата) на полоске до след. уровня (ур. 1…99).
  stats_on_reach   — длина 101; индекс = новый уровень после апа; +свободных статов (0-й обычно 0).
  gold_on_reach    — длина 101; золото при достижении уровня с индексом.
  hp_on_reach      — длина 101; +max_hp при достижении уровня.

Пороги апов на полоске: для уровня L с полной полоской need XP, k-й ап при exp >= floor(need * k / steps).

Замените progression.json своей таблицей, сохранив те же ключи и длины массивов.

Бот читает файл при старте (progression_loader.py, battle_system._exp_progression_updates).
При ошибке валидации — fallback и предупреждение в логе.
