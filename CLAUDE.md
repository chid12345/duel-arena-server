## Регламент для ИИ — Duel Arena Server

Короткий, без воды. Это **единственный** регламент, который читается всегда.
Детали и справка — в `docs/*` (читать только по теме задачи). **Старт:** `docs/INDEX.md` — выбрать блок по ключевым словам, не «всё подряд».

## Роль ИИ
- ИИ = геймдизайн + архитектура + качество.
- Любая идея → “пакет решения”: **зачем игроку / баланс / активность / прогресс**. Если 2+ пункта пустые — не делаем или упрощаем.

## Законы проекта (обязательны)

### Закон 1. Лимит размера файла (200 / 300)
- **Норма**: до **200** строк.
- **Если файл > 200 (обычно 200–250)**: новую логику **не добавляем** — начинаем новый модуль или делим файл.
- **Аварийный порог**: **300** строк → **сразу** распил и только потом новые фичи.
- **Исключений по размеру нет**; монолиты — только пакеты (`db_schema/`, `battle_system/`) с модулями ≤200 строк.

### Закон 2. Структура “Лего” (новый дом)
- Новая тема/механика/контент → **сразу отдельный модуль** (файл/папка).
- Если файл раздувается по смыслам → “Матрёшка” (папка). Если раздувается из‑за списка контента/умений → “Библиотека умений”.

### Закон 3. Импорты без циклов
Разрешённая цепочка:
`config/progression_loader → db_core → db_schema → repositories → database → battle_system → handlers/api_server`.
Запрещены импорты “вверх по цепочке”. Если надо “кверх” — передаём параметром.

### Закон 4. Минимальный контекст (правило 3 файлов)
Начинаем с **1–3 файлов** по теме задачи. Расширяем только если не хватает данных.

### Закон 5. Отчёт после работы (1 строка)
`Готово: <что сделано>. version.py X→Y, GAME_VERSION A→B. Запушено.`

### Закон 6. Безопасный распил (чеклист)
- План распила (3–5 строк).
- Сначала перенос (без изменения логики), потом улучшения.
- Импорты привести к Закону 3.
- Smoke-тест: `python -c "from database import db; from handlers import BotHandlers"`.
- Проект рабочий сразу, без “доделаем потом”.

### Закон 7. Коротко и по делу
Меняем **только нужное** для задачи. Не пишем простыни, не делаем косметику без причины.

### Закон 8. Нет мёртвого кода
Не оставляем “старое” закомментированным “про запас”. Если не используется — удаляем (или сначала проверяем использования).

### Закон 9. Новая категория → новый дом
Новая категория контента/системы → отдельный модуль. Не “подселять” в чужой файл.

### Закон 10. Версии + коммит + пуш (после каждой правки кода)
После **каждой** правки кода:
- поднять `version.py`
- поднять `GAME_VERSION` на **+0.01**
- `git commit`
- `git push`

## Где лежат подробности (9 блоков + навигатор)
- Навигатор по ключевым словам → `docs/INDEX.md`
- Архитектура/карта/куда класть/SQL/Mixin → `docs/ARCHITECTURE.md`
- Бой/лог боя/правила изменений → `docs/BATTLE.md`
- Экономика/баланс/синки → `docs/ECONOMY.md`
- Магазин/предметы/зелья → `docs/SHOP.md`
- UX/тексты/кнопки → `docs/UX_TEXTS.md`
- Квесты/ежедневки/ивенты (старые) → `docs/QUESTS.md`
- Система заданий v2/стрик/достижения → `docs/TASKS.md`
- Кланы/соц/рефералы/платежи → `docs/SOCIAL.md`
- Мировой босс/рейды/WS-бродкаст/свитки рейда → `docs/WORLD_BOSS.md`
