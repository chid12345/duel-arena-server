# WORLD_BOSS — Мировой босс (рейды толпой)

Справка. Читать только когда задача про **мирового босса**: рейды, общий HP, WebSocket-бродкаст, свитки воскрешения, рейд-свитки, награды рейда, анонсы в чат.

Не путать с обычными PvP/PvE боями — те в `docs/BATTLE.md`.

---

## Короткие правила фичи

| Параметр | Значение |
|---|---|
| Расписание спавна | 6 раз/день: 00:00, 04:00, 08:00, 12:00, 16:00, 20:00 (UTC) |
| Длительность боя | 10 минут |
| HP босса | `500 × онлайн_игроков`, минимум 10 000 |
| Босс универсальный | да, с рандомом ±10% статов (сила/ловкость/интуиция) |
| Имя босса | одно из 10 вариантов, рандом при спавне |
| Ответка босса | каждые 6 сек, 8% HP случайного из **топ-5 по урону** |
| Коронные удары | 75% HP → -3% всем, 50% → -5%, 25% → -8% |
| Окно уязвимости | раз в 60 сек, длится 5 сек, урон **x3** |
| Награды победы | +200% (база × вклад%) |
| Награды поражения | 30% (утешительные) |
| Сундук «последний удар» | за золото (обычный) |
| Сундук «больше всех урона» | за алмазы (редкий) |
| Забор наград | вручную через UI после конца рейда |
| Смерть в рейде | выбыл до конца рейда, остальную игру играешь |
| Воскрешение | только через свитки из магазина (30/60/100% HP) |

## Таблицы БД

| Таблица | Что хранит |
|---|---|
| `world_boss_spawns` | Один рейд = одна запись. HP, имя, статы-профиль (JSON), статус, победители сундуков |
| `world_boss_hits` | Каждый удар игрока (damage, crit, окно-уязвимости) — для подсчёта вклада |
| `world_boss_player_state` | HP игрока в рейде + 2 слота активных рейд-свитков + `is_dead` |
| `world_boss_rewards` | Непрочитанные награды (gold/exp/diamonds/сундук, `claimed`) |
| `inventory` | Свитки воскрешения (`item_type='resurrection_scroll'`) и рейд-свитки (`item_type='raid_scroll'`) |
| `players.wb_reminder_opt_in` | Флаг «напомнить за 5 мин» |

**Важно:** HP игрока в рейде хранится в `world_boss_player_state.current_hp` — **не трогать** `players.current_hp` (это HP для обычных дуэлей).

## Свитки (item_name / item_type)

### Свитки воскрешения (обычный магазин, 10 зарядов в пачке)

| item_name | Эффект | Цена |
|---|---|---|
| `res_30` | Воскрешение с 30% HP | золото |
| `res_60` | Воскрешение с 60% HP | алмазы |
| `res_100` | Воскрешение со 100% HP | алмазы (больше 60%) |

### Рейд-свитки (новая вкладка «Рейд-свитки», 10 зарядов, 1 заряд = 1 рейд)

| item_name | Эффект (только в рейде) |
|---|---|
| `damage_25` | +25% урон |
| `defense_20` | +20% защита |
| `dodge_10` | +10% уворот |
| `power_10` | +10% сила / доп. урон |
| `crit_10` | +10% крит |

**Правила рейд-свитков:**
- Работают **только** в бою с мировым боссом. В обычных боях **игнорируются** (не тратятся).
- Одновременно активно **максимум 2** (слоты `raid_scroll_1`, `raid_scroll_2` в `world_boss_player_state`).
- Применяются **вручную** (кнопкой «Применить» в UI рейда) → списывается 1 заряд → действует до конца рейда.
- **Суммируются** с обычными эликсирами (не заменяют их).

## Формулы

### HP босса
```
HP = max(10_000, 500 * online)
```

### Рандом-статы босса (при спавне)
```json
{"str": 0.9..1.1, "agi": 0.9..1.1, "int": 0.9..1.1}
```
Множители от базовых статов босса. Игрок **не знает** расклад заранее.

### Удар игрока по боссу (сервер считает сам — анти-чит)
```
damage = base_damage(player.stats) * raid_buffs(slots) * (x3 если в окне уязвимости) * (x1.5..2.0 если крит)
```

### Ответка босса игроку (каждые 6 сек)
```
target = random из TOP5 по total_damage
dmg_to_player = 8% * player.max_hp * boss.stat_profile.str / (1 + player.defense_mult)
```

### Коронный удар (при порогах HP босса)
```
75% → all_players.hp -= 3% max_hp
50% → all_players.hp -= 5% max_hp  + screen shake
25% → all_players.hp -= 8% max_hp  + эффект «Хаос» 10 сек
```

### Награды
```
если победа:
  gold_reward = base_gold * 2.0 * contribution_pct     # +200% базы, пропорционально вкладу
  exp_reward  = base_exp  * 2.0 * contribution_pct
если поражение:
  gold_reward = base_gold * 0.3 * contribution_pct
  exp_reward  = base_exp  * 0.3 * contribution_pct
```

## REST API (план)

| Метод | URL | Что делает |
|---|---|---|
| GET | `/api/world_boss/state` | Текущее состояние рейда + таймер следующего + незабранные награды |
| POST | `/api/world_boss/hit` | Удар по боссу (анти-чит: кулдаун 300 мс, сервер считает урон) |
| POST | `/api/world_boss/use_scroll` | Применить рейд-свиток (item_name, slot 1/2) |
| POST | `/api/world_boss/resurrect` | Воскрешение (item_name = res_30/60/100) |
| POST | `/api/world_boss/claim_reward` | Забрать награду после рейда |
| POST | `/api/world_boss/reminder_toggle` | Включить/выключить «🔔 Напомни за 5 мин» |

## WebSocket

| Канал | Сообщения |
|---|---|
| `/ws/world_boss` | Тик раз в 1 сек: `{hp, online, top3, last_hits[]}`. События: `crown_hit`, `vulnerability_window`, `boss_dead`, `spawn_started`, `spawn_ended` |

## UI вкладка «⚔️ Босс»

3 состояния:
- **Ожидание** — таймер до спавна, кнопка «🔔 Напомни», история прошлого рейда
- **Бой** — HP-бар, спрайт босса, кнопка «УДАРИТЬ», аватарки «бьющих рядом», топ-3
- **Итоги** — победа/поражение, твой вклад в %, кнопка «Забрать награду», «Поделиться» (Фаза 2)

Вход: нижнее меню, 6-я кнопка. Также — по клику «Вступить» из анонса бота в чате.

## Задание + достижения

- **Дейлик:** «Удар по боссу 1 раз за день» (сброс 00:00 UTC).
- **Достижения:** 1 / 5 / 10 / 100 / 1000 побед в рейде (участник победившей команды).

## Что НЕ делаем в MVP

- Heatmap ударов (куда бьют) — переусложнение.
- Скролл к голове босса — ломает мобильный UX.
- Эффект «тереть иней» — дорого, однодневка.
- `/buff @username` через Reply — Фаза 3.
- Общий сундук с «бросить кубик» — Фаза 3.
- Emoji Status участника — Фаза 3.
- Типы боссов (ледяной/огненный) — Фаза 2. В MVP **один универсальный** с рандом-статами.
- Динамический фон по HP — Фаза 2.
- Вторая стадия босса на 50% HP — Фаза 2.

## Чеклист при правках системы

1. Миграция БД → `db_schema/sqlite_migrations_part_world_boss.py`.
2. Запросы к БД → `repositories/world_boss/` (новый миксин).
3. Константы (HP, ответка, коронные удары, список 10 имён) → `config/world_boss/`.
4. Серверный расчёт урона — **только сервер** (никогда не доверять клиенту).
5. Атомарность last-hit: `UPDATE ... WHERE current_hp > 0 RETURNING current_hp`.
6. Рестарт сервера посреди рейда — состояние в БД, не в памяти.
7. Рейд-свиток вне рейда — **игнор, не тратить**.
8. Версии + коммит + пуш (Закон 10).
